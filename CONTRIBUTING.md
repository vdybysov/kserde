# Contributing to kserde

Thank you for your interest in contributing!

## How to contribute

1. **Fork** the repository on GitHub
2. **Clone** your fork locally
3. **Create a branch** for your changes (`git checkout -b feature/your-feature` or `bugfix/your-fix`)
4. **Make your changes** and ensure tests pass: `./gradlew :example:test`
5. **Commit** with a clear message
6. **Push** to your fork and open a **Pull Request**

## Development setup

```bash
git clone https://github.com/vdybysov/kserde.git
cd kserde
./gradlew build
```

## Running tests

```bash
./gradlew :example:test
```

## Code style

- Follow Kotlin coding conventions
- Keep changes focused and reasonably sized
- Add tests for new functionality

## Pull request guidelines

- Describe what the PR does and why
- Reference any related issues
- Ensure CI passes
